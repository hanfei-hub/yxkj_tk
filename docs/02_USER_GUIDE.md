# 使用说明文档

## 1. 启动方式

### 启动前端

在项目根目录执行：

```powershell
.\start_desktop.ps1
```

桌面程序会直接打开主窗口。开发阶段不要求先登录，所有菜单都会展示。

### 启动本地后端

如需本地后端调试，在项目根目录执行：

```powershell
.\start_backend.ps1
```

本地后端默认地址：

```text
http://127.0.0.1:8000
```

线上后端当前地址：

```text
http://120.26.207.89:8000
```

## 2. 登录说明

主窗口左下角显示当前用户信息。

- 未登录时显示开发预览模式。
- 点击左下角登录按钮，会弹出登录小窗。
- token 失效时会提示“请重新登录”，主页面仍应可打开。

默认账号：

| 角色 | 用户名 | 密码 |
| --- | --- | --- |
| 管理员 | admin | admin123 |
| 老师 | teacher | teacher123 |
| 学生 | student | student123 |

## 3. 管理员操作

### 3.1 配置大模型

进入“模型配置”菜单。

填写字段：

- 配置名称：例如 `豆包快速模型`、`DeepSeek V1`。
- 服务商：可填写 `doubao`、`deepseek`、`openai`、`custom`。
- API Base URL：模型接口根地址。
- API Key：模型密钥。
- 模型名称：平台提供的模型 ID。
- 温度：建议翻译和衍生品生成使用较低温度，例如 `0.2`。
- 最大 tokens：衍生品生成建议配置较大，例如 `12000`。
- 启用状态：启用。
- 默认模型：点击“设为默认”。

说明：

- FastMoss 标题翻译和衍生品生成都会优先使用默认可用模型。
- 翻译不需要深度思考模式。
- 衍生品生成不需要深度思考模式。

### 3.2 配置 FastMoss

进入“第三方 API”菜单。

FastMoss 配置要求：

- `service_type` 使用 `fastmoss`。
- API 地址填写 FastMoss 文档中的接口地址或根地址。
- Access Key 填写 FastMoss 接口密钥。
- 状态设置为启用。

同步规则：

- 同步日本区新品榜。
- 跨境商品：是。
- 全托管商品：否。
- 日期固定向前 4 天。

### 3.3 配置 1688 API

进入“第三方 API”菜单。

1688 配置要求：

- `service_type` 使用 `1688_api`。
- API Base URL 填写第三方提供的 1688 API 根地址。
- Access Key / Secret Key 按第三方要求填写。
- 状态设置为启用。

如果第三方 API 字段名和默认字段不同，可以在备注中填写 JSON 适配参数，例如：

```json
{
  "search_path": "/search",
  "method": "POST",
  "keyword_field": "keyword",
  "page_field": "page",
  "page_size_field": "page_size",
  "items_path": "data.items",
  "total_path": "data.total"
}
```

### 3.4 同步 FastMoss 商品

管理员进入对应页面后点击“同步 FastMoss”。

同步成功后：

- 旧 FastMoss 商品会被清除。
- 新商品入库。
- 标题会调用模型翻译成中文。
- 图片 URL、价格、销量、类目、原始数据会保存。
- 后续会触发衍生品生成。

### 3.5 查看选品属性

进入“选品属性”菜单。

当前系统固定使用 8 个选品维度：

1. 使用场景
2. 商品周期性
3. 目标群体
4. 短视频流量种草适配能力
5. 日本市场偏好
6. 是否属于新奇特商品
7. 复购属性
8. 竞品属性

这 8 个维度也是老师拒绝原因和权重学习的基础。

## 4. 老师操作

### 4.1 查看教师看板

进入“教师看板”菜单。

页面展示 FastMoss 原商品：

- 商品图片。
- 商品名称。
- 价格。
- 销量。
- 查看衍生品按钮。

### 4.2 审核衍生品

点击某个原商品的“查看衍生品”。

审核窗口展示：

- 当前原商品标题。
- 该原商品下的衍生品列表。
- 每个衍生品左侧图片。
- 右侧 8 个维度分析小卡片。
- 分数和当前状态。
- 拒绝按钮。

### 4.3 拒绝衍生品

点击“拒绝”后会弹出拒绝原因窗口。

操作步骤：

1. 在下拉框中选择拒绝维度。
2. 可填写备注说明。
3. 提交拒绝。

提交后：

- 衍生品状态改为 rejected。
- 老师审核记录写入数据库。
- 对应商品分族的该维度权重会降低。
- 审核窗口不会关闭，可以继续审核其他衍生品。

### 4.4 切换下一个原商品

审核窗口底部有“下一个原商品”按钮。

点击后会切换到下一个原商品的衍生品列表，方便连续批改。

## 5. 学生操作

学生进入“智能选品”菜单。

可使用：

- AI 选品对话。
- 衍生品推荐展示。
- 新品榜单展示。

学生不能查看老师审核通过后的内部数据，也不能访问后台配置。

## 6. 常见问题

### 前端弹出 invalid token

含义：登录 token 已失效或本地保存的 token 不可用。

处理：

- 关闭提示。
- 点击左下角重新登录。

系统要求是：即使 token 失效，主页面也应继续加载。

### 图片加载慢或显示不出来

可能原因：

- 本地网络代理问题。
- 图片源站访问慢。
- FastMoss 返回的图片 URL 失效。

处理：

- 优先检查数据库中的图片 URL 是否正常。
- 如果是本地网络问题，不在本机反复修代理，直接确认网络异常。

### 同步 FastMoss 提示无效 token

可能原因：

- FastMoss Key 配置错误。
- 第三方 API 配置未启用。
- 请求日期或接口参数不符合 FastMoss 要求。

处理：

- 检查第三方 API 配置。
- 检查后端同步日志。

### 模型调用很慢

可能原因：

- 使用了深度思考模型。
- max_tokens 太大。
- 模型账户余额不足。
- 服务器到模型 API 网络慢。

建议：

- 翻译使用快速模型。
- 衍生品生成使用默认快速模型，不开启深度思考。
- 温度设置较低。
