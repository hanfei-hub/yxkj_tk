# 产品需求文档

## 1. 产品概述

TK 日本跨境智能选品系统是一套面向 TikTok 日本站跨境电商的智能选品工具。产品最终形态为 Windows 桌面程序，前端使用 PySide6，后端部署在 Linux 云服务器。

系统以 FastMoss 日本区新品榜数据为输入，通过大模型完成标题翻译、商品分族、衍生品生成和分析报告沉淀，再由老师对衍生品方向进行人工审核。审核数据会反向沉淀到商品分族维度权重中，用于后续模型生成更符合业务判断的衍生品。

## 2. 产品目标

- 自动采集日本区 TikTok 电商新品榜商品。
- 将日文商品标题翻译成中文，并保留原始图片、价格、销量等信息。
- 基于原商品标题和图片，用大模型生成 10 个衍生品方向。
- 每个衍生品沉淀 8 个固定分析维度。
- 老师可以对衍生品进行拒绝批改，并选择拒绝维度。
- 系统根据老师批改数据调整商品分族维度权重。
- 后续同类商品再次生成衍生品时，将历史权重传给大模型，提高选品质量。
- 衍生品生成后，通过 1688 API 搜索候选货源，并用大模型基于“衍生品名称 + 1688 首图”判断匹配度，达到阈值后写入货源信息。

## 3. 用户角色

### 管理员

管理员负责系统配置和基础数据维护。

主要权限：

- 用户管理。
- 模型配置。
- 第三方 API 配置。
- 选品属性维护。
- FastMoss 数据同步。
- 查看任务看板和同步状态。

### 老师

老师负责审核 AI 生成的衍生品方向。

主要权限：

- 查看教师看板中的原商品。
- 点击原商品进入衍生品审核窗口。
- 查看每个衍生品的图片、标题、分数、状态和 8 个维度分析。
- 对不符合要求的衍生品执行拒绝操作。
- 拒绝时选择对应维度原因。
- 批改结果进入数据库，影响后续权重。

### 学生

学生负责使用智能选品能力，但不能看到老师审核通过后的内部结果。

主要权限：

- 使用 AI 智能选品对话。
- 查看前端展示的商品推荐数据。
- 不可访问教师审核数据和后台配置页面。

## 4. 核心功能

### 4.1 智能选品

页面内容：

- AI 选品输入框。
- 衍生品推荐展示区。
- 新品榜单展示区。

当前要求：

- 商品卡展示图片、商品名称、价格、销量。
- 价格统一使用日元符号表现，页面中使用 `円`。
- 图片需要自适应商品卡尺寸，避免裁剪不一致。
- AI 推荐商品后续改为展示衍生品。

### 4.2 教师看板

页面内容：

- 原商品商品卡。
- 每行 6 个商品。
- 商品卡展示方式与智能选品商品卡一致，额外保留“查看衍生品”操作。
- 点击原商品后进入该原商品对应的衍生品审核窗口。

### 4.3 衍生品审核

审核对象是 AI 生成的衍生品方向，不是具体 1688 商品。

审核窗口要求：

- 左侧预留图片位置。
- 右侧展示 8 个维度小卡片，从左到右排列。
- 不展示“通过”按钮。
- 点击“拒绝”后弹出拒绝原因窗口。
- 拒绝原因下拉框使用 8 个固定维度。
- 拒绝后不关闭审核窗口。
- 底部提供“下一个原商品”按钮，用于切换到下一个原商品的衍生品列表。

固定 8 个维度：

1. 使用场景
2. 商品周期性
3. 目标群体
4. 短视频流量种草适配能力
5. 日本市场偏好
6. 是否属于新奇特商品
7. 复购属性
8. 竞品属性

### 4.4 管理后台

后台以桌面程序菜单形式提供。

主要菜单：

- 用户管理。
- 模型配置。
- 第三方 API。
- 选品属性。
- 主题皮肤。
- 任务看板。

开发阶段要求：

- 所有菜单均展示。
- 未登录也能进入主窗口。
- 左下角显示当前用户信息。
- 未登录时点击登录按钮弹出登录小窗。
- 登录 token 失效时提示“请重新登录”，不能导致主页面无法加载。

### 4.5 模型配置

管理员可以配置多个大模型。

需要支持：

- OpenAI 兼容接口。
- 豆包。
- DeepSeek。
- 其他可兼容 `/chat/completions` 的模型。

关键字段：

- 配置名称。
- 服务商。
- API Base URL。
- API Key。
- 模型名称。
- 温度。
- 最大输出 tokens。
- 是否默认。
- 启用状态。

当前规则：

- FastMoss 标题翻译和衍生品生成都使用默认可用模型。
- 翻译可使用快速模型，不需要深度思考模式。
- 衍生品生成也不需要深度思考模式。

### 4.6 FastMoss 数据同步

FastMoss 同步要求：

- 查询日本区新品榜。
- 是否跨境商品：是。
- 是否全托管商品：否。
- 日期固定向前 4 天。
- 每次入库前清除旧 FastMoss 商品数据。
- 入库字段包括商品标题、中文标题、图片 URL、价格、销量、类目、店铺、排名、原始数据。
- 价格使用日元。
- 销量使用个数展示。
- 同步完成后触发衍生品生成。

定时要求：

- 每天 8:30 执行 FastMoss 同步。
- FastMoss 同步完成后，自动触发衍生品生成。
- 衍生品生成可异步执行，不阻塞前端。
- 可以一条原商品一条原商品生成衍生品。

### 4.7 衍生品生成

输入：

- FastMoss 原商品中文标题。
- FastMoss 原商品图片 URL。
- 选品逻辑提示词。
- 当前商品分族历史维度权重。

输出：

- 每个原商品生成 10 个衍生品。
- 每个衍生品包含标题、说明、推荐理由、人群、场景、风险、评分、8 个维度分析报告。
- 衍生品必须绑定原商品 ID 和商品分族 ID。

当前大模型选品逻辑：

- 基于商品标题和图片分析相关衍生品。
- 遵守日本地区禁售、禁运、物流限制和平台规则。
- 输出 8 个固定维度的完整分析。
- 没有依据的内容需要标注“无充足依据，仅作参考线索”。

### 4.8 商品分族与权重

问题背景：

单纯用原商品 ID 作为权重维度不合理，因为相同商品方向可能在不同 FastMoss 数据中反复出现。

解决方案：

- FastMoss 商品入库时，根据标题和类目提取关键词。
- 将商品归入商品分族。
- 分族代表稳定商品方向，例如“捏捏乐硅胶玩具”和“捏捏乐水晶玩具”应归为相近分族。
- 老师审核后，维度权重写到商品分族维度权重表。
- 后续同分族商品生成衍生品时，优先读取该分族权重。

权重规则：

- 8 个维度初始权重相同，各为 12.5%。
- 老师拒绝某个维度时，该维度权重下降。
- 其他维度按比例归一化。
- 后续模型生成时，将权重百分比传入提示词。

### 4.9 1688 商品匹配

衍生品刚生成时只有名称和分析报告，缺少真实货源图片、价格、销量等信息。

正式闭环：

1. 根据衍生品名称或搜索关键词调用 1688 API。
2. 分批分页获取候选商品，最多可查 200 个。
3. 每批候选只把“衍生品名称 + 1688 首图 URL”交给大模型判断。
4. 大模型返回匹配度和一句话原因。
5. 当匹配度达到阈值，例如 90%，立即选中该 1688 商品。
6. 停止继续搜索。
7. 将 1688 商品图片、价格、销量、店铺、链接、匹配报告写回衍生品表。

## 5. 非功能需求

- Windows 桌面端默认全屏打开。
- 前端图片加载不能卡死主界面。
- 前端网络失败时不能导致主窗口无法显示。
- token 失效提示“请重新登录”。
- 后端 token 过期时间为 1 周。
- 后端修改后需要部署到服务器并重启服务。
- 前端单独修改时可直接运行源码 UI，不必每次重新打包。

## 6. 验收标准

- 管理员能配置模型和第三方 API。
- 管理员能同步 FastMoss 数据。
- FastMoss 商品能展示图片、中文标题、日元价格、销量。
- 每个原商品能生成 10 个衍生品。
- 衍生品能展示 8 个维度分析。
- 老师拒绝衍生品后，拒绝记录入库。
- 老师选择的拒绝维度能影响商品分族权重。
- 后续同分族商品生成衍生品时，提示词包含历史权重。
- 1688 API 配置完成后，系统能自动匹配货源并写回衍生品。
